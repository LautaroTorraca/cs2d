============================================================
        PROTOCOLO DE COMUNICACIÓN - COUNTER STRIKE 2D
============================================================

1. ESTRUCTURA GENERAL DEL MENSAJE
----------------------------------

Todos los mensajes siguen el siguiente formato binario, codificados en Big Endian:

    +----------+----------+-------------------------------------+
    | Campo    | Tamaño   | Descripción                         |
    +----------+----------+-------------------------------------+
    | Opcode   | 1 byte   | Código que identifica el mensaje    |
    | Longitud | 2 bytes  | Tamaño del contenido (payload)      |
    | Contenido| Variable | Payload específico del mensaje      |
    +----------+----------+-------------------------------------+

    - Todos los valores numéricos se codifican en BIG ENDIAN.
    - Todos los mensajes son BINARIOS.

------------------------------------------------------------
2. MENSAJES CLIENTE → SERVIDOR (SERIALIZACIÓN)
------------------------------------------------------------

2.1 LOBBY PRINCIPAL

    +--------+---------------------+----------------------------------------+
    | Opcode | Nombre              | Descripción                            |
    +--------+---------------------+----------------------------------------+
    | 0x01   | CREATE_GAME         | Crear una nueva partida                |
    | 0x02   | JOIN_GAME           | Unirse a una partida existente         |
    | 0x03   | GAME_LIST_REQUEST   | Solicitar lista de partidas            |
    | 0x09   | DISCONNECT          | Desconectarse del servidor             |
    +--------+---------------------+----------------------------------------+

    +--------+---------------------+----------------------------------------+
